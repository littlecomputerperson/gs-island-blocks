Class Overview

About GS_OGLDisplay

The GS_OGLDisplay class manages the OpenGL rendering context and provides high-level control over the graphics pipeline. It handles context creation, viewport setup, rendering state management, and provides utilities for both 2D and 3D rendering modes.

This class is the foundation of GameSystem's graphics system, abstracting OpenGL initialization and providing a clean interface for configuring rendering features like anti-aliasing, VSync, depth testing, alpha blending, and lighting.

Key Features

  • OpenGL Context Management: Create and destroy rendering contexts on Windows and SDL2 platforms
  • Viewport Configuration: Set up perspective projection for 3D and orthographic projection for 2D
  • Anti-Aliasing: Enable smooth line and polygon rendering
  • VSync Control: Synchronize rendering with monitor refresh rate
  • Alpha Blending: Support transparency in textures and sprites
  • Depth Testing: Proper 3D object occlusion
  • Lighting System: Configurable ambient, diffuse, and specular lighting
  • 2D/3D Mode Switching: Easy transition between rendering modes
  • Resolution Scaling: Support for upscaling and downscaling with aspect ratio preservation

Rendering Modes

GS_OGLDisplay supports two primary rendering modes:

3D Mode (Perspective Projection)

  • Used for 3D scenes with depth
  • Perspective projection with configurable field of view
  • Depth testing enabled
  • Supports lighting and camera positioning
  • Activated with BeginRender() / EndRender()

2D Mode (Orthographic Projection)

  • Used for sprites, fonts, menus, and UI
  • Orthographic projection matching screen dimensions
  • Depth testing typically disabled
  • Coordinate system: (0,0) at bottom-left, Y increases upward
  • Activated with BeginRender2D() / EndRender2D()

Resolution Scaling

The class provides static methods for resolution-independent rendering:

  • Scale Factors: Multiply sprite coordinates to support different resolutions
  • Render Modifiers: Offset coordinates for letterboxing/pillarboxing
  • Aspect Ratio Preservation: Maintain correct proportions when upscaling/downscaling

Internal Resolution System

GameSystem uses an internal resolution (typically 960x540) that gets scaled to the actual window size. This allows games to render at a fixed resolution while supporting multiple display sizes:

// Game renders at 960x540 internally
// Window runs at 1920x1080 (or any size)
// Scale factors: 2.0x, 2.0x
// Sprites automatically scaled to fit

This approach simplifies game logic by working with consistent coordinates regardless of actual window size.

Usage Pattern

Typical workflow for using GS_OGLDisplay:

// In GameInit()
m_gsDisplay.Create(GetDevice(), GetWidth(), GetHeight(), GetColorDepth());
m_gsDisplay.EnableVSync(TRUE);
m_gsDisplay.EnableBlending(TRUE);

// In GameLoop() - 2D Rendering
glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
m_gsDisplay.BeginRender2D(GetWindow());

// Render sprites, fonts, UI
m_sprite.Render();
m_font.Render();

m_gsDisplay.EndRender2D();
SwapBuffers(GetDevice());

// In GameShutdown()
m_gsDisplay.Destroy();

Initialization Methods

Create()

File: gs_ogl_display.cpp

Creates the OpenGL rendering context and initializes default rendering states.

Signature (Windows):

BOOL Create(HDC& hDC, int nWidth, int nHeight, int nDepth)

Signature (SDL2):

BOOL Create(SDL_Window* pWindow, int nWidth, int nHeight, int nDepth)

Parameters:

  • hDC (Windows) - Device context handle from the window
  • pWindow (SDL2) - SDL window pointer
  • nWidth - Viewport width in pixels
  • nHeight - Viewport height in pixels
  • nDepth - Color depth in bits (32 recommended, ignored on SDL2)

Behavior:

  • If already created, calls Destroy() first
  • Windows: Sets up pixel format descriptor, chooses pixel format, creates GL context with wglCreateContext()
  • SDL2: Creates GL context with SDL_GL_CreateContext()
  • Activates the rendering context
  • Calls SetViewport() to configure projection
  • Enables texture mapping (GL_TEXTURE_2D)
  • Disables face culling
  • Enables depth testing with GL_LEQUAL
  • Sets smooth shading
  • Sets perspective correction hint to GL_NICEST
  • Sets clear color to black (0, 0, 0, 0)
  • Sets clear depth to 1.0

Returns:

  • TRUE - Context created successfully
  • FALSE - Creation failed (invalid handle, pixel format failure, etc.)

Example:

if (!m_gsDisplay.Create(GetDevice(), GetWidth(), GetHeight(), 32))
{
    GS_Error::Report(__FILE__, __LINE__, "Failed to create OpenGL display!");
    return FALSE;
}

// Configure display settings
m_gsDisplay.EnableVSync(TRUE);
m_gsDisplay.EnableBlending(TRUE);

File: gs_ogl_display.cpp

GLvoid Destroy()

SetViewport()

File: gs_ogl_display.cpp

Configures the OpenGL viewport and sets up perspective projection for 3D rendering.

Signature:

GLvoid SetViewport(GLsizei glWidth, GLsizei glHeight, GLdouble glView)

Parameters:

  • glWidth - Viewport width in pixels
  • glHeight - Viewport height in pixels
  • glView - Field of view angle in degrees (typically 45.0)

Behavior:

  • Returns immediately if display not ready
  • Prevents divide-by-zero by ensuring height â‰¥ 1
  • Sets viewport with glViewport(0, 0, width, height)
  • Switches to projection matrix mode
  • Loads identity matrix
  • Sets up perspective with gluPerspective():
    • Field of view: glView degrees
    • Aspect ratio: width / height
    • Near clipping plane: 0.1
    • Far clipping plane: 100.0
  • Switches to modelview matrix mode
  • Loads identity matrix

Returns:

  • None (void)

Example:

// Set viewport with 45-degree field of view
m_gsDisplay.SetViewport(800, 600, 45.0);

// Wider field of view for more peripheral vision
m_gsDisplay.SetViewport(1920, 1080, 90.0);

Note: This method is called automatically by Create(). You typically only call it manually if you need to change the viewport or field of view after initialization.

Display Configuration Methods

EnableAliasing()

File: gs_ogl_display.cpp

Enables or disables anti-aliasing for smoother line and polygon rendering.

Signature:

GLvoid EnableAliasing(BOOL bEnable = TRUE, GLenum glType = GL_NICEST)

Parameters:

  • bEnable - TRUE to enable, FALSE to disable (default: TRUE)
  • glType - Quality hint: GL_NICEST (best quality), GL_FASTEST (best performance), or GL_DONT_CARE (default: GL_NICEST)

Behavior:

  • If enabled, sets line smooth hint to glType
  • If enabled, sets polygon smooth hint to glType
  • If disabled, sets both hints to GL_FASTEST
  • Updates m_bIsAliasingEnabled flag

Returns:

  • None (void)

Example:

// Enable with highest quality
m_gsDisplay.EnableAliasing(TRUE, GL_NICEST);

// Disable for better performance
m_gsDisplay.EnableAliasing(FALSE);

// Enable with balanced quality
m_gsDisplay.EnableAliasing(TRUE, GL_DONT_CARE);

Performance Impact: Anti-aliasing can reduce frame rate, especially on older hardware. Consider making this a user-configurable option.

EnableVSync()

File: gs_ogl_display.cpp

Enables or disables vertical synchronization to eliminate screen tearing.

Signature:

GLvoid EnableVSync(BOOL bEnable = TRUE)

Parameters:

  • bEnable - TRUE to enable VSync, FALSE to disable (default: TRUE)

Behavior:

  • Windows: Uses wglGetProcAddress("wglSwapIntervalEXT") to get extension function
  • Windows: Calls wglSwapIntervalEXT(1) to enable or wglSwapIntervalEXT(0) to disable
  • SDL2: Calls SDL_GL_SetSwapInterval(1) to enable or SDL_GL_SetSwapInterval(0) to disable
  • Updates m_bIsVSyncEnabled flag

Returns:

  • None (void)

Example:

// Enable VSync to prevent tearing
m_gsDisplay.EnableVSync(TRUE);

// Disable for higher frame rates
m_gsDisplay.EnableVSync(FALSE);

// Toggle VSync based on setting
m_gsDisplay.EnableVSync(m_settings.bEnableVSync);

Note: VSync caps frame rate to the monitor's refresh rate (typically 60 FPS). Disabling VSync allows unlimited frame rate but may cause screen tearing.

EnableDepthTesting()

File: gs_ogl_display.cpp

Enables or disables depth testing for proper 3D object occlusion.

Signature:

GLvoid EnableDepthTesting(BOOL bEnable = TRUE, GLenum glType = GL_LEQUAL)

Parameters:

  • bEnable - TRUE to enable depth testing, FALSE to disable (default: TRUE)
  • glType - Depth comparison function: GL_NEVER, GL_LESS, GL_EQUAL, GL_LEQUAL, GL_GREATER, GL_NOTEQUAL, GL_GEQUAL, GL_ALWAYS (default: GL_LEQUAL)

Behavior:

  • If enabled, sets depth function with glDepthFunc(glType)
  • If disabled, calls glDisable(GL_DEPTH_TEST)

Returns:

m_gsDisplay.EnableDepthTesting(FALSE);

Note: Depth testing is automatically managed by BeginRender2D() and EndRender2D(). Manual control is rarely needed.

EnableBlending()

File: gs_ogl_display.cpp

Enables or disables alpha blending for transparency effects.

Signature:

GLvoid EnableBlending(BOOL bEnable = TRUE, GLenum glSourceFactor = GL_SRC_ALPHA, 
                   GLenum glDestFactor = GL_ONE_MINUS_SRC_ALPHA)

Parameters:

  • bEnable - TRUE to enable blending, FALSE to disable (default: TRUE)
  • glSourceFactor - Source blend factor (default: GL_SRC_ALPHA)
  • glDestFactor - Destination blend factor (default: GL_ONE_MINUS_SRC_ALPHA)

Behavior:

  • If enabled, calls glBlendFunc(glSourceFactor, glDestFactor)
  • If enabled, calls glEnable(GL_BLEND)
  • If disabled, calls glDisable(GL_BLEND)

Common Blend Modes:

Source Factor Dest Factor Effect
GL_SRC_ALPHA GL_ONE_MINUS_SRC_ALPHA Standard alpha transparency
GL_SRC_ALPHA GL_ONE Additive blending (light effects)
GL_ZERO GL_SRC_COLOR Multiplicative blending (darken)

Returns:

  • None (void)

Example:

// Standard alpha transparency
m_gsDisplay.EnableBlending(TRUE);

// Additive blending for glowing effects
m_gsDisplay.EnableBlending(TRUE, GL_SRC_ALPHA, GL_ONE);

// Disable blending
m_gsDisplay.EnableBlending(FALSE);

Note: Blending must be enabled for transparent textures (TGA with alpha channel) to render correctly.

EnableLighting()

File: gs_ogl_display.cpp

Enables or disables OpenGL lighting system.

Signature:

GLvoid EnableLighting(BOOL bEnable = TRUE, GLenum glLightNum = GL_LIGHT1)

Parameters:

  • bEnable - TRUE to enable lighting, FALSE to disable (default: TRUE)
  • glLightNum - Light source to use: GL_LIGHT0 through GL_LIGHT7 (default: GL_LIGHT1)

Behavior:

  • If enabled, calls glEnable(GL_LIGHTING)
  • If enabled, calls glEnable(glLightNum) to activate specific light
  • If disabled, calls glDisable(GL_LIGHTING)

Returns:

  • None (void)

Example:

// Enable lighting with light 1
m_gsDisplay.EnableLighting(TRUE, GL_LIGHT1);

// Configure light properties
m_gsDisplay.SetLightAmbient(0.2f, 0.2f, 0.2f, 1.0f);
m_gsDisplay.SetLightDiffuse(1.0f, 1.0f, 1.0f, 1.0f);

// Disable lighting for 2D rendering
m_gsDisplay.EnableLighting(FALSE);

Note: Lighting is automatically disabled during 2D rendering mode (BeginRender2D()) and restored afterward.

BeginRender2D()

File: gs_ogl_display.cpp

Begins 2D orthographic rendering mode for sprites, fonts, and UI.

Signature:

BOOL BeginRender2D(HWND& hWnd)

Parameters:

  • hWnd - Window handle (HWND on Windows, SDL_Window* on SDL2)

Behavior:

  • Returns FALSE if display not ready or invalid window handle
  • Gets client rectangle dimensions
  • Saves current depth testing state, then disables it
  • Saves current lighting state, then disables it
  • Switches to projection matrix and pushes current matrix
  • Loads identity matrix
  • Sets up orthographic projection with glOrtho():
    • Left: 0
    • Right: client width
    • Bottom: 0 (bottom of screen)
    • Top: client height
    • Near/Far: Based on larger dimension for rotation capability
  • Switches to modelview matrix and pushes current matrix

Coordinate System:

  • Origin (0,0) at bottom-left corner
  • X increases rightward
  • Y increases upward (OpenGL convention)
  • Matches sprite rendering coordinate system

Returns:

  • TRUE - 2D mode activated successfully
  • FALSE - Failed (display not ready or invalid window)

Example:

glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
m_gsDisplay.BeginRender2D(GetWindow());

// Render 2D content
m_backgroundSprite.Render();
m_playerSprite.Render();
m_font.Render();

m_gsDisplay.EndRender2D();
SwapBuffers(GetDevice());

EndRender2D()

File: gs_ogl_display.cpp

Ends 2D rendering mode and restores previous rendering state.

Signature:

BOOL EndRender2D()

Parameters:

  • None

Behavior:

  • Switches to projection matrix
  • Pops saved projection matrix (restores perspective)
  • Switches to modelview matrix
  • Pops saved modelview matrix
  • Restores depth testing state if it was enabled
  • Restores lighting state if it was enabled

Returns:

  • TRUE - Always returns TRUE

Example:

m_gsDisplay.EndRender2D();

Note: Always pair BeginRender2D() with EndRender2D() to properly restore rendering state.

Resolution Scaling Overview

These static methods control the global scaling system that allows games to render at a fixed internal resolution while supporting multiple window sizes. The scaling system automatically adjusts sprite coordinates and sizes.

How Scaling Works

Internal Resolution: 960x540
Window Resolution:   1920x1080
Scale Factor:        2.0x, 2.0x
Render Offset:       0, 0

Sprite at (100, 100) renders at:
X = (100 * 2.0) + 0 = 200
Y = (100 * 2.0) + 0 = 200

Platform-Specific Notes

Windows Implementation

  • Uses HGLRC for rendering context
  • Uses ChoosePixelFormat() and SetPixelFormat()
  • Creates context with wglCreateContext()
  • VSync via wglSwapIntervalEXT extension
  • Buffer swap via SwapBuffers(HDC)

SDL2 Implementation

  • Uses SDL_GLContext for rendering context
  • Pixel format handled automatically by SDL
  • Creates context with SDL_GL_CreateContext()
  • VSync via SDL_GL_SetSwapInterval()
  • Buffer swap via SDL_GL_SwapWindow()

Cross-Platform Compatibility

The class provides identical functionality on all platforms through conditional compilation. The only visible difference is the handle type passed to methods (HDC vs SDL_Window*).
