Class Overview

About GS_OGLMenu

The GS_OGLMenu class provides a complete, customizable menu system for GameSystem applications. It combines a tiled sprite-based menu background with bitmap font rendering to create professional-looking menus with full keyboard and mouse support.

This class is essential for game UIs, providing an easy way to create title screens, pause menus, options screens, and any other menu-based interface. The menu automatically handles layout, highlighting, navigation, and rendering.

Key Features

  • Tiled Menu Background: Uses a 16-tile sprite set for borders, separators, and backgrounds
  • Bitmap Font Integration: Renders menu title and options using GS_OGLFont
  • Optional Title: Supports menus with or without a title section
  • Highlight System: Visual feedback for selected options with keyboard/mouse navigation
  • Dynamic Options: Add, remove, or modify options at runtime
  • Custom Colors: Separate colors for normal and highlighted text
  • Auto-Sizing: Menu dimensions automatically adjust to content
  • Scaling Support: Scale entire menu uniformly
  • Collision Detection: Get bounding rectangles for mouse interaction
  • Cross-Platform: Works identically on Windows, Linux, and macOS

Usage Pattern

Typical workflow for using GS_OGLMenu:

// Create menu
GS_OGLMenu menu;
menu.Create("data/menu_01.tga", fontTexture.GetID(), 
            fontTexture.GetWidth(), fontTexture.GetHeight(),
            16, 16, 16);

// Configure menu
menu.SetTitle("MAIN MENU");
menu.AddOption("NEW GAME");
menu.AddOption("LOAD GAME");
menu.AddOption("OPTIONS");
menu.AddOption("QUIT");
menu.SetHighlight(0);

// Position menu
menu.SetDestXY((screenWidth - menu.GetWidth()) / 2,
               (screenHeight - menu.GetHeight()) / 2);

// In game loop - handle input
if (keyPressed == GSK_UP)   menu.HighlightPrev();
if (keyPressed == GSK_DOWN) menu.HighlightNext();
if (keyPressed == GSK_ENTER)
{
    int selected = menu.GetHighlight();
    // Process selection...
}

// Render menu
m_gsDisplay.BeginRender2D(GetWindow());
menu.Render(normalColor, highlightColor);
m_gsDisplay.EndRender2D();

Initialization Methods

Create() - From Files

File: gs_ogl_menu.cpp

Creates a menu by loading both menu and font textures from files.

Signature:

BOOL Create(const char* pszMenuPathname, const char* pszFontPathname, 
            int nTilesPerLine, int nTileWidth, int nTileHeight,
            BOOL bFiltered = FALSE, BOOL bMipmap = TRUE)

Parameters:

  • pszMenuPathname - Path to menu tile texture (TGA/PCX)
  • pszFontPathname - Path to font texture (TGA/PCX)
  • nTilesPerLine - Characters per line in font (e.g., 16)
  • nTileWidth - Width of each tile/character in pixels
  • nTileHeight - Height of each tile/character in pixels
  • bFiltered - TRUE for bilinear filtering (default: FALSE)
  • bMipmap - TRUE to generate mipmaps (default: TRUE)

Behavior:

  • Creates menu sprite with 16 tiles
  • Creates font with matching tile size
  • Font tile size must match menu tile size
  • Sets m_bIsReady = TRUE on success

Returns:

  • TRUE - Menu created successfully
  • FALSE - Creation failed

Example:

GS_OGLMenu menu;
if (!menu.Create("data/menu.tga", "data/font.tga", 16, 16, 16))
{
    GS_Error::Report(__FILE__, __LINE__, "Failed to create menu!");
    return FALSE;
}

Create() - From Existing Texture

File: gs_ogl_menu.cpp

Creates a menu using an existing OpenGL texture ID for the font.

Signature:

BOOL Create(const char* pszMenuPathname, GLuint glFontTexture, 
            int nWidth, int nHeight, int nTilesPerLine,
            int nTileWidth, int nTileHeight,
            BOOL bFiltered = FALSE, BOOL bMipmap = TRUE)

Parameters:

  • pszMenuPathname - Path to menu tile texture
  • glFontTexture - OpenGL texture ID for font
  • nWidth - Font texture width in pixels
  • nHeight - Font texture height in pixels
  • nTilesPerLine - Characters per line in font
  • nTileWidth - Width of each tile/character
  • nTileHeight - Height of each tile/character
  • bFiltered - Texture filtering
  • bMipmap - Generate mipmaps for menu

Returns:

  • TRUE - Menu created successfully
  • FALSE - Creation failed

Example:

GS_OGLMenu menu;

            fontTexture.GetWidth(), fontTexture.GetHeight(),
            16, 16, 16);

Use Case: Efficient when multiple menus or fonts share the same texture.

Rendering Methods

Render() - Basic

File: gs_ogl_menu.cpp

Renders the menu with default white text color.

Signature:

BOOL Render(HWND hWnd = NULL)

Parameters:

  • hWnd - Window handle (ignored, for compatibility)

Behavior:

  • Returns FALSE if menu not created
  • Renders menu borders (corners, edges)
  • Renders menu background fill
  • Renders title separator if title exists
  • Renders highlight background if option selected
  • Renders title text (if present)
  • Renders all option texts
  • Uses white (1.0, 1.0, 1.0, 1.0) for all text

Returns:

  • TRUE - Menu rendered successfully
  • FALSE - Rendering failed

Render() - With Colors

File: gs_ogl_menu.cpp

Renders the menu with custom colors for normal and highlighted text.

Signature:

BOOL Render(GS_OGLColor gsNormalColor, GS_OGLColor gsHighlightColor, HWND hWnd = NULL)

Parameters:

  • gsNormalColor - Color for non-highlighted options
  • gsHighlightColor - Color for highlighted option
  • hWnd - Window handle (ignored)

Behavior:

  • Same as basic Render() for menu background
  • Title rendered in white (1.0, 1.0, 1.0, 1.0)
  • Normal options rendered in gsNormalColor
  • Highlighted option rendered in gsHighlightColor
  • Automatically switches colors based on highlight state

Returns:

  • TRUE - Menu rendered successfully
  • FALSE - Rendering failed

Example:

// Semi-transparent gray for normal text

GS_OGLColor highlightColor(1.0f, 1.0f, 1.0f, 1.0f);

menu.Render(normalColor, highlightColor);

// Yellow highlight
menu.Render(GS_OGLColor(0.7f, 0.7f, 0.7f, 1.0f),  // Gray normal
            GS_OGLColor(1.0f, 1.0f, 0.0f, 1.0f)); // Yellow highlight

Parameters:

        break;
    }
}

Menu Navigation with Keyboard

int nKey = m_gsKeyboard.GetBufferedKey();

switch (nKey)
{
    case GSK_UP:
        menu.HighlightPrev();
        m_gsSound.PlaySample(SOUND_MENU_MOVE);
        break;
        
    case GSK_DOWN:
        menu.HighlightNext();
        m_gsSound.PlaySample(SOUND_MENU_MOVE);
        break;
        
    case GSK_ENTER:
        int selected = menu.GetHighlight();
        m_gsSound.PlaySample(SOUND_MENU_SELECT);
        
        switch (selected)
        {
            case 0: m_nGameState = NEW_GAME; break;
            case 1: m_nGameState = LOAD_GAME; break;
            case 2: m_nGameState = OPTIONS; break;
            case 3: m_nGameState = QUIT; break;
        }
        break;
        
    case GSK_ESCAPE:
        m_nGameState = TITLE_SCREEN;
        break;
}

Menu Navigation with Mouse

// Mouse hover detection
static int oldMouseX = 0, oldMouseY = 0;
int mouseX = m_gsMouse.GetX();
int mouseY = m_gsMouse.GetY();

if (mouseX != oldMouseX || mouseY != oldMouseY)
{
    // Check each option for mouse hover
    for (int i = 0; i < menu.GetNumOptions(); i++)
    {
        RECT optionRect;
        menu.GetOptionRect(i, &optionRect);
        
        if (m_gsCollide.IsCoordInRect(mouseX, mouseY, optionRect))
        {
            menu.SetHighlight(i);
            m_gsSound.PlaySample(SOUND_MENU_MOVE);
            break;
        }
    }
    
    oldMouseX = mouseX;
    oldMouseY = mouseY;
}

// Mouse click detection
static bool mouseReleased = true;

if (!m_gsMouse.IsLeftPressed())
{
    mouseReleased = true;
}

if (m_gsMouse.IsLeftPressed() && mouseReleased)
{
    RECT highlightRect;
    menu.GetHighlightRect(&highlightRect);
    
    if (m_gsCollide.IsCoordInRect(mouseX, mouseY, highlightRect))
    {
        int selected = menu.GetHighlight();
        // Process selection...
        mouseReleased = false;
    }
}

GS_OGLColor highlightColor(1.0f, 1.0f, 1.0f, menuAlpha);

menu.Render(normalColor, highlightColor);

Best Practices

  • Create Once: Initialize menus in GameInit(), reuse throughout
  • Update Dynamically: Use SetOption() to update text without recreating menu
  • Clear Before Rebuild: Call ClearOptions() before adding new options
  • Center Menus: Calculate position based on menu dimensions
  • Keyboard First: Always support keyboard navigation
  • Mouse as Bonus: Mouse support is convenient but keyboard should be primary
  • Sound Feedback: Play sounds on navigation and selection
  • Consistent Tile Size: Menu tiles and font characters must be same size

💡 Pro Tip

Create a reusable menu texture template with borders, backgrounds, and highlights that matches your game's art style. You can then use the same menu texture for all menus in your game, just changing the font and options.

Platform-Specific Notes

Cross-Platform Compatibility

GS_OGLMenu is completely cross-platform and works identically on all platforms:

  • Windows - Full support with native OpenGL
  • Linux - Full support via SDL2 + OpenGL
  • macOS - Full support via SDL2 + OpenGL

There are no platform-specific implementations or behavior differences.

Performance Considerations

  • Rendering: Menu rendering is efficient - draws individual tiles only when needed
  • Memory: Each option string is dynamically allocated, minimal overhead
  • Updates: Changing option text requires memory reallocation
  • Scaling: Scaling is free (GPU handles it)

Texture Requirements

  • Menu Texture: Must contain exactly 16 tiles in specific order
  • Font Texture: Character grid (typically 16×16 = 256 characters)
  • Tile Alignment: Font character size must match menu tile size
  • Format: TGA with alpha channel recommended

✅ Design Tip

When creating menu tiles, design them to tile seamlessly. The background tile (index 11) should tile perfectly both horizontally and vertically. Test your menu with different option counts to ensure it looks good at any size.
