Class Overview

About GS_Application

The GS_Application class is the core foundation of the GameSystem framework. It provides a complete application framework for creating games by managing the window, OpenGL context, message loop, frame timing, and platform abstraction.

Your game inherits from this class and overrides virtual methods to implement custom behavior:

class GS_Demo : public GS_Application
{
public:
    BOOL GameInit() override;
    BOOL GameLoop() override;
    BOOL GameShutdown() override;
    // ... other overrides
};

Platform Implementations

GS_Application has two platform-specific implementations that share the same interface:

File Platform Description
gs_app.cpp Windows Native Windows API implementation with menu bar and accelerator support
gs_app_sdl.cpp Linux / macOS SDL2 implementation for cross-platform compatibility

The implementation is selected at compile-time using the GS_PLATFORM_WINDOWS preprocessor define. Both implementations provide the same API, ensuring your game code works identically on all platforms.

Key Features

  • Cross-Platform: Abstracts Windows API and SDL2 for seamless multi-platform support
  • Frame Rate Management: Built-in frame timing and FPS control
  • Window Management: Handles window creation, resizing, and fullscreen/windowed mode switching
  • Message Handling: Processes keyboard, mouse, and system messages
  • Focus Management: Automatically handles window activation/deactivation
  • Virtual Method Pattern: Clean separation of framework and game logic
  • Unified Interface: Same code works on Windows, Linux, and macOS

Platform Support

Platform Backend Features
Windows Native Windows API Menu bar, accelerator keys, native controls
Linux SDL2 X11/Wayland support, keyboard controls
macOS SDL2 Intel and Apple Silicon support

SDL2 Implementation (gs_app_sdl.cpp)

Overview

The gs_app_sdl.cpp file provides a complete SDL2-based implementation of GS_Application for Linux and macOS platforms. This implementation maintains API compatibility with the Windows version while using SDL2's cross-platform windowing and event systems.

Platform Detection

The SDL2 implementation is conditionally compiled using:

#ifndef GS_PLATFORM_WINDOWS
    // SDL2 implementation
#endif

SDL2-Specific Features

Window Creation

SDL2 window creation differs from Windows:

  • Uses SDL_CreateWindow() instead of Windows CreateWindowEx()
  • Creates an OpenGL context with SDL_GL_CreateContext()
  • Supports both windowed and fullscreen desktop modes
  • Automatically centers window on screen
// SDL2 window flags
Uint32 flags = SDL_WINDOW_OPENGL | SDL_WINDOW_SHOWN;
if (!m_bIsWindowed) {
    flags |= SDL_WINDOW_FULLSCREEN;
}

m_pWindow = SDL_CreateWindow(
    "Game System OpenGL",
    SDL_WINDOWPOS_CENTERED,
    SDL_WINDOWPOS_CENTERED,
    m_nDisplayWidth,
    m_nDisplayHeight,
    flags
);

Event Handling

SDL2 uses an event-based system instead of Windows messages:

SDL Event Windows Message Purpose
SDL_QUIT WM_QUIT Application exit requested
SDL_WINDOWEVENT_FOCUS_LOST WM_ACTIVATEAPP(FALSE) Window lost focus
SDL_WINDOWEVENT_FOCUS_GAINED WM_ACTIVATEAPP(TRUE) Window gained focus
SDL_KEYDOWN WM_KEYDOWN Key pressed
SDL_KEYUP WM_KEYUP Key released

Main Loop Differences

The SDL2 Run() implementation:

  • Uses SDL_PollEvent() instead of PeekMessage()
  • Processes SDL events and converts them to Windows-compatible messages
  • Calls GameRelease()/GameRestore() on focus changes
  • Handles Alt+Enter for fullscreen toggle via SDL keyboard events
  • Uses SDL_GL_SwapWindow() instead of SwapBuffers()
INT GS_Application::Run()
{
    SDL_Event event;
    bool running = true;

    while (running) {
        // Process all pending events
        while (SDL_PollEvent(&event)) {
            switch (event.type) {
                case SDL_QUIT:
                    running = false;
                    break;
                    
                case SDL_WINDOWEVENT:
                    // Handle focus changes
                    if (event.window.event == SDL_WINDOWEVENT_FOCUS_LOST) {
                        if (m_bIsActive && m_bIsReady) {
                            GameRelease();
                            m_bIsActive = FALSE;
                        }
                    }
                    // ...
                    break;
            }
        }

        // Render frame
        if (m_bIsActive && m_bIsReady) {
            if ((m_fFrameTime = m_gsFrameTimer.GetFrameTime()) >= m_fWaitTime) {
                m_gsFrameTimer.MarkFrame();
                if (TRUE != GameLoop()) {
                    running = false;
                }
                SDL_GL_SwapWindow(m_pWindow);
            }
        }
    }

    this->Destroy();
    return 0;
}

OpenGL Context Management

SDL2 manages the OpenGL context differently:

  • Creation: SDL_GL_CreateContext() creates the GL context
  • Attributes: Set via SDL_GL_SetAttribute() before window creation
  • VSync: Controlled with SDL_GL_SetSwapInterval(1)
  • Buffer Swap: SDL_GL_SwapWindow() instead of SwapBuffers()
  • Cleanup: SDL_GL_DeleteContext() on destruction

Fullscreen Mode

SDL2 fullscreen works differently than Windows:

  • Uses SDL_WINDOW_FULLSCREEN or SDL_WINDOW_FULLSCREEN_DESKTOP
  • Desktop fullscreen maintains current resolution (recommended)
  • True fullscreen changes display mode like Windows
  • No need for ChangeDisplaySettings() - SDL handles it

Platform Abstraction

The SDL2 implementation provides platform-neutral types:

// Windows types → SDL2 equivalents
HWND      → SDL_Window*
HDC       → SDL_Window* (reused for compatibility)
HINSTANCE → NULL (not applicable)

// Accessor methods work identically:
GetWindow()   // Returns SDL_Window*
GetDevice()   // Returns SDL_Window* (used for SwapBuffers)
GetInstance() // Returns NULL on SDL2 platforms

Initialization Sequence (SDL2)

  1. SDL_Init(SDL_INIT_VIDEO | SDL_INIT_EVENTS) - Initialize SDL
  2. SDL_GL_SetAttribute() - Configure OpenGL context attributes
  3. SDL_CreateWindow() - Create game window
  4. SDL_GL_CreateContext() - Create OpenGL context
  5. SDL_GL_SetSwapInterval(1) - Enable VSync
  6. GameInit() - Initialize game resources
  7. Enter main event loop

Shutdown Sequence (SDL2)

  1. GameShutdown() - Clean up game resources
  2. SDL_GL_DeleteContext() - Destroy OpenGL context
  3. SDL_DestroyWindow() - Destroy window
  4. SDL_Quit() - Shutdown SDL subsystems

Compatibility Notes

⚠️ Platform Differences

  • No Menu Bar: SDL2 doesn't support native menu bars (use in-game menus instead)
  • No Accelerators: Alt+X, Alt+Enter handled manually via keyboard events
  • Different Focus Behavior: SDL_WINDOWEVENT vs WM_ACTIVATEAPP
  • Message Box: Uses SDL_ShowSimpleMessageBox() instead of MessageBox()
  • Cursor Management: Uses SDL_ShowCursor() and SDL_SetCursor()

Testing Your Game on SDL2

To ensure your game works correctly on SDL2 platforms:

  • Test on both Windows and Linux/macOS to verify cross-platform behavior
  • Don't rely on Windows-specific features (menu bars, accelerators)
  • Use keyboard shortcuts that work with SDL2's event system
  • Test fullscreen mode transitions thoroughly
  • Verify focus loss/gain handling (alt-tab behavior)
  • Ensure frame timing works correctly on different platforms

Core Methods

Create()

Files: gs_app.cpp (Windows), gs_app_sdl.cpp (SDL2)

Creates and initializes the application window, sets up the display mode, and prepares the application for rendering.

Signature:

BOOL Create(int nWidth, int nHeight, int nDepth, BOOL bIsWindowed)

Parameters:

  • nWidth - Display width in pixels (must be > 0)
  • nHeight - Display height in pixels (must be > 0)
  • nDepth - Color depth in bits (8, 16, 24, or 32)
  • bIsWindowed - TRUE for windowed mode, FALSE for fullscreen

Behavior:

  • Uses provided dimensions or defaults to GS_DEFAULT_WIDTH × GS_DEFAULT_HEIGHT
  • In fullscreen mode, uses current screen resolution instead of provided dimensions
  • Windows: Registers window class, creates window with CreateWindowEx()
  • SDL2: Initializes SDL, sets GL attributes, creates window with SDL_CreateWindow()
  • Creates OpenGL context
  • Calls GameInit() to initialize game resources
  • Starts frame timer

Returns:

  • TRUE - Application created successfully
  • FALSE - Creation failed (check error reports)

Example:

if (!Create(800, 600, 32, TRUE))
{
    // Handle error
}

SetMode()

Files: gs_app.cpp (Windows), gs_app_sdl.cpp (SDL2)

Changes the display mode by destroying and recreating the application with new settings.

Signature:

BOOL SetMode(int nWidth, int nHeight, int nDepth, BOOL bIsWindowed)

Parameters:

  • nWidth - New display width in pixels
  • nHeight - New display height in pixels
  • nDepth - New color depth in bits
  • bIsWindowed - TRUE for windowed, FALSE for fullscreen

Behavior:

  • If not yet created, calls Create()
  • Saves previous display settings
  • Destroys current application
  • Attempts to create with new settings
  • If failure, reverts to previous settings
  • Game resources are reloaded via GameInit()

Returns:

  • TRUE - Mode change successful
  • FALSE - Mode change failed (reverted to previous settings)

Example:

// Toggle between windowed and fullscreen
SetMode(1024, 768, 32, !IsWindowed());

Destroy()

Files: gs_app.cpp (Windows), gs_app_sdl.cpp (SDL2)

Destroys the application window and releases all associated resources.

Signature:

void Destroy()

Parameters:

  • None

Behavior:

  • Calls GameShutdown() for custom cleanup
  • Sets status flags to inactive/not ready
  • Hides the window
  • Windows: Restores desktop settings (if fullscreen), releases DC, destroys window, unregisters class
  • SDL2: Destroys GL context with SDL_GL_DeleteContext(), destroys window with SDL_DestroyWindow(), calls SDL_Quit()

Returns:

  • None (void)

Note: This is called automatically by the destructor.

Run()

Files: gs_app.cpp (Windows), gs_app_sdl.cpp (SDL2)

The main application loop that processes messages and runs the game.

Signature:

INT Run()

Parameters:

  • None

Behavior (Windows):

  • Calculates wait time based on target frame rate
  • Processes window messages with PeekMessage()/GetMessage()
  • Translates keyboard accelerators (if enabled)
  • During idle time, runs GameLoop() at the specified frame rate
  • Waits for minimum time between frames to maintain target FPS
  • Continues until WM_QUIT message is received
  • Calls Destroy() on exit

Behavior (SDL2):

  • Calculates wait time based on target frame rate
  • Processes SDL events with SDL_PollEvent()
  • Converts SDL events to Windows-compatible messages for MsgProc()
  • Handles window focus changes automatically
  • Runs GameLoop() when active and ready
  • Swaps buffers with SDL_GL_SwapWindow()
  • Continues until SDL_QUIT event received
  • Calls Destroy() on exit

Returns:

  • INT - Exit code (0 for normal exit)

Example:

int main()
{
    GS_Demo game;
    if (game.Create(800, 600, 32, TRUE))
    {
        return game.Run();
    }
    return 1;
}

GameLoop()

File: gs_demo.cpp (your implementation)

Called every frame. This is where your main game logic runs.

Signature:

virtual BOOL GameLoop()

Parameters:

  • None

When Called:

  • Every frame when application is active and not paused
  • Respects target frame rate set by SetFrameRate()

Typical Structure:

  1. Process input (keyboard, mouse)
  2. Update game logic (physics, AI, collisions)
  3. Render frame (sprites, UI, effects)
  4. Swap buffers to display

Returns:

  • TRUE - Continue running
  • FALSE - Exit application

Example:

BOOL GS_Demo::GameLoop()
{
    // Input
    if (m_gsKeyboard.IsKeyPressed(GSK_ESCAPE))
        return FALSE;
    
    // Update
    m_player.Update();
    
    // Render
    glClear(GL_COLOR_BUFFER_BIT);
    m_gsDisplay.BeginRender2D(GetWindow());
    m_player.Render();
    m_gsDisplay.EndRender2D();
    SwapBuffers(GetDevice());
    
    return TRUE;
}

MsgProc()

File: gs_demo.cpp (your implementation)

Processes Windows messages or SDL events. Override to handle custom input and window messages.

Signature:

virtual LRESULT MsgProc(HWND hWnd, UINT uMsg, WPARAM wParam, LPARAM lParam)

Parameters:

  • hWnd - Window handle (or SDL_Window* cast to HWND)
  • uMsg - Message identifier (WM_* constants)
  • wParam - Message-specific parameter
  • lParam - Message-specific parameter

Common Messages:

  • WM_KEYDOWN / WM_KEYUP - Keyboard input
  • WM_CHAR - Character input (for text)
  • WM_MOUSEMOVE - Mouse movement
  • WM_LBUTTONDOWN / WM_LBUTTONUP - Mouse buttons
  • WM_COMMAND - Menu commands (Windows only)

Returns:

  • LRESULT - Message-specific return value
  • Call GS_Application::MsgProc() for default processing

Example:

LRESULT GS_Demo::MsgProc(HWND hWnd, UINT uMsg, WPARAM wParam, LPARAM lParam)
{
    switch (uMsg)
    {
    case WM_KEYDOWN:
        m_gsKeyboard.KeyDown(wParam);
        m_gsKeyboard.AddKeyToBuffer(wParam);
        break;
    case WM_KEYUP:
        m_gsKeyboard.KeyUp(wParam);
        break;
    case WM_MOUSEMOVE:
        m_gsMouse.SetX(LOWORD(lParam));
        m_gsMouse.SetY(GetHeight() - HIWORD(lParam));
        break;
    }
    
    return GS_Application::MsgProc(hWnd, uMsg, wParam, lParam);
}

File: gs_app.cpp

void SetFrameRate(float fFrameRate)
