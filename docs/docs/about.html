Overview

GameSystem (GS) is a cross-platform game development framework built with OpenGL and SDL2, designed for creating 2D and 3D games. This comprehensive library provides a complete foundation for game development with built-in state management, rendering systems, audio playback, input handling, and resource management.

The framework follows an object-oriented design pattern with platform abstraction layers, allowing developers to write games once and deploy them across Windows, Linux, and macOS with minimal platform-specific code changes.

What is GameSystem?

GameSystem is a C++ game development library that abstracts the complexity of OpenGL rendering, SDL2 window management, and cross-platform compatibility. It provides:

  • A complete application framework with state-based game flow
  • 2D sprite rendering with effects (rotation, scaling, transparency, animation)
  • Tile-based map system for level design
  • Bitmap font rendering system
  • Collision detection utilities
  • Particle effect systems
  • Menu and UI components
  • Audio playback for music and sound effects
  • Input handling for keyboard and mouse
  • Configuration persistence via INI files
  • Frame timing and FPS management

Tech Stack

Core Technologies

  • C++ - C++98/C++11 compatible codebase
  • OpenGL 2.1 - Fixed-function pipeline graphics rendering
  • SDL2 - Cross-platform window management, event handling, and input
  • SDL2_mixer - Audio playback for music (MP3) and sound effects (WAV)
  • CMake 3.10+ - Cross-platform build configuration
  • GCC/Clang/MSVC - Compiler support across all platforms

Framework Features

  • State Machine Architecture: Built-in game state management (intro, menu, gameplay, pause, game over)
  • Platform Abstraction: Unified codebase with seamless Windows API and SDL2 compatibility
  • OpenGL Rendering: Object-oriented C++ wrappers around OpenGL for sprites, textures, and fonts
  • 2D Sprite System: Basic and animated sprite support with transforms and effects
  • Tile Map Engine: Grid-based level design with texture mapping
  • Audio Integration: Music streaming and sound effect playback via SDL2_mixer
  • Input Management: Buffered and real-time keyboard/mouse input handling
  • Collision Detection: 2D bounding box and pixel-perfect collision utilities
  • Menu System: Interactive menu components with keyboard and mouse navigation
  • Particle Effects: Built-in particle system for visual effects
  • Settings Persistence: INI file-based configuration storage
  • Frame Rate Control: VSync support and frame timing utilities
  • Resource Management: Automatic texture and sprite cleanup
  • Aspect Ratio Scaling: Resolution-independent rendering with letterboxing

Design Philosophy

GameSystem emphasizes:

  • Simplicity: Easy-to-understand object-oriented design
  • Portability: Write once, run on Windows, Linux, and macOS
  • Performance: Direct OpenGL access with minimal overhead
  • Flexibility: Extend the framework to suit your game's needs
  • Completeness: All essential game systems included out of the box
