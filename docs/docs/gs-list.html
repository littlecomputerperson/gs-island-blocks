Class Overview

About GS_List

The GS_List template class provides a dynamic, type-safe array container for GameSystem applications. It's a C++ template that can store any data type (primitives, pointers, objects, or structures) in a dynamically resizable array with automatic memory management.

This class serves as a lightweight alternative to std::vector, providing essential list operations with a simple, easy-to-understand interface. It's particularly useful for managing collections of game objects, textures, sounds, or other resources.

Key Features

  • Template-Based: Works with any data type (int, float, pointers, structs, classes)
  • Dynamic Resizing: Automatically grows as items are added
  • Type Safety: Compile-time type checking ensures correct usage
  • Memory Management: Automatic allocation and deallocation
  • Array-Style Access: Use [] operator like standard arrays
  • Flexible Operations: Add, insert, remove, find items easily
  • Efficient Storage: Contiguous memory layout for cache efficiency
  • Safe Bounds Checking: Assertions prevent out-of-bounds access

Template Declaration

template<class type>
class GS_List
{
    // ...
};

Member Variables

All member variables are private:

Variable Type Description
m_ptArray type* Pointer to dynamically allocated array
m_nNumItems int Number of items currently in the list
m_nNumAllocated int Number of items for which memory is allocated

Usage Examples

Creating Lists of Different Types

// List of integers
GS_List<int> scores;

// List of floats
GS_List<float> velocities;

// List of pointers to textures
GS_List<GS_OGLTexture*> textures;

// List of sprite objects
GS_List<GS_OGLSprite> sprites;

// List of FMOD sound modules
GS_List<FMUSIC_MODULE*> musicList;

Basic Operations

// Create a list of integers
GS_List<int> numbers;

// Add items
numbers.AddItem(10);
numbers.AddItem(20);
numbers.AddItem(30);

// Access items
int first = numbers[0];        // 10
int second = numbers.GetItem(1); // 20

// Find item
int index = numbers.FindItem(20); // Returns 1

// Remove item
numbers.RemoveItem(20);

// Check size
int count = numbers.GetNumItems(); // 2

// Clear all
numbers.RemoveAllItems();

Common Use Cases

  • Resource Management: Store lists of textures, sounds, or sprites
  • Game Objects: Manage collections of enemies, bullets, or particles
  • Level Data: Store tile maps, waypoints, or spawn locations
  • UI Components: Manage menu items or buttons
  • Audio Playlists: Store music tracks or sound effects

Constructor & Destructor

GS_List()

File: gs_list.h

Constructor that initializes an empty list with optional pre-allocation.

Signature:

GS_List(int size = 0)

Parameters:

  • size - Number of items to pre-allocate space for (default: 0)

Initialization:

  • Sets m_ptArray to NULL
  • Sets m_nNumItems to 0
  • Sets m_nNumAllocated to 0
  • If size > 0, calls SetNumItems() to pre-allocate

Examples:

// Empty list, no pre-allocation
GS_List<int> scores;

// Pre-allocate space for 100 items
GS_List<float> particles(100);

// List of texture pointers
GS_List<GS_OGLTexture*> textures(10);

~GS_List()

File: gs_list.h

Destructor that frees all allocated memory.

Behavior:

  • Deletes the internal array if allocated
  • Automatically called when list goes out of scope
  • Note: Does NOT delete objects pointed to by pointers in the list

Important: If storing pointers to dynamically allocated objects, you must delete them before the list is destroyed:

// Clean up pointer list
for (int i = 0; i < textures.GetNumItems(); i++)
{
    delete textures[i];
}
textures.RemoveAllItems();

Add Methods

AddItem()

File: gs_list.h

Adds an item to the end of the list. Automatically resizes if necessary.

Signature:

void AddItem(type tData)

Parameters:

  • tData - Item to add to the list

Behavior:

  • If no space allocated, allocates space for 1 item
  • If list is full, allocates space for 1 additional item
  • Appends item to the end of the list
  • Increments item count

Examples:

GS_List<int> numbers;
numbers.AddItem(10);
numbers.AddItem(20);
numbers.AddItem(30);
// List now contains: [10, 20, 30]

GS_List<GS_OGLTexture*> textures;
GS_OGLTexture* pTexture = new GS_OGLTexture();
textures.AddItem(pTexture);

InsertItem()

File: gs_list.h

Inserts an item at a specific position in the list.

Signature:

void InsertItem(int nIndex, type tData)

Parameters:

  • nIndex - Position to insert at (0-based)
  • tData - Item to insert

Behavior:

  • Returns without action if index is invalid
  • If index equals list size, calls AddItem()
  • Otherwise, shifts all items from index onward one position forward
  • Inserts new item at the specified position
  • Increments item count

Example:

GS_List<int> numbers;
numbers.AddItem(10);
numbers.AddItem(30);
numbers.InsertItem(1, 20);
// List now contains: [10, 20, 30]

Remove Methods

RemoveItem()

File: gs_list.h

Removes all items matching the specified value from the list.

Signature:

void RemoveItem(const type tData)

Parameters:

  • tData - Item value to remove

Behavior:

  • Searches the entire list for matching items
  • Removes ALL occurrences of the item
  • Uses == operator for comparison
  • Does not resize allocated memory

Example:

GS_List<int> numbers;
numbers.AddItem(10);
numbers.AddItem(20);
numbers.AddItem(10);
numbers.RemoveItem(10);
// List now contains: [20]

RemoveIndex()

File: gs_list.h

Removes the item at a specific index.

Signature:

void RemoveIndex(int nIndex)

Parameters:

  • nIndex - Index of item to remove (0-based)

Behavior:

  • Returns without action if index is invalid
  • Shifts all items after the removed item one position backward
  • Decrements item count
  • Uses memmove() for efficient memory shifting

Example:

GS_List<int> numbers;
numbers.AddItem(10);
numbers.AddItem(20);
numbers.AddItem(30);
numbers.RemoveIndex(1);
// List now contains: [10, 30]

RemoveEmptyItems()

File: gs_list.h

Removes all items that equal zero or NULL from the list.

Signature:

void RemoveEmptyItems()

Parameters:

  • None

Behavior:

  • Iterates backwards through the list
  • Removes all items that equal 0 (for numbers) or NULL (for pointers)
  • Uses == operator comparison with 0

Example:

GS_List<int*> pointers;
pointers.AddItem(new int(10));
pointers.AddItem(NULL);
pointers.AddItem(new int(20));
pointers.RemoveEmptyItems();
// List now contains only non-NULL pointers

RemoveAllItems()

File: gs_list.h

Clears the entire list and frees all allocated memory.

Signature:

void RemoveAllItems()

Parameters:

  • None

Behavior:

  • Sets item count to 0
  • Deletes the internal array
  • Sets allocated count to 0
  • List becomes completely empty

Example:

GS_List<int> numbers;
numbers.AddItem(10);
numbers.AddItem(20);
numbers.RemoveAllItems();
// List is now empty, memory freed

Find Methods

FindItem()

File: gs_list.h

Searches for an item in the list and returns its index.

Signature:

int FindItem(const type tData)

Parameters:

  • tData - Item to search for

Behavior:

  • Performs linear search from start to end
  • Uses == operator for comparison
  • Returns index of FIRST matching item found

Returns:

  • Index of item (0-based) if found
  • -1 if item not found

Examples:

GS_List<int> numbers;
numbers.AddItem(10);
numbers.AddItem(20);
numbers.AddItem(30);

int index = numbers.FindItem(20);  // Returns 1
int notFound = numbers.FindItem(99); // Returns -1

// Check if item exists
if (numbers.FindItem(20) != -1)
{
    // Item is in the list
}

Get/Set Methods

operator[]

File: gs_list.h (inline)

Array-style subscript operator for accessing items (const version).

Signature:

inline type operator [] (int nIndex) const

Parameters:

  • nIndex - Index of item to access (0-based)

Behavior:

  • Asserts if index is out of bounds (debug builds)
  • Returns item at specified index

Returns:

  • Copy of the item at the specified index

Example:

GS_List<int> numbers;
numbers.AddItem(10);
numbers.AddItem(20);

int first = numbers[0];   // 10
int second = numbers[1];  // 20

GetNumItems()

File: gs_list.h (inline)

Returns the number of items currently in the list.

inline int GetNumItems()

Parameters:

  • None

Returns:

  • Number of items in the list

Example:

SetNumItems()

File: gs_list.h

Resizes the list to a specific number of items.

Signature:

void SetNumItems(int nNumToAllocate)

Parameters:

  • nNumToAllocate - New size for the list

Behavior:

  • If size is 0, deletes the array entirely
  • Otherwise, allocates space for the specified number of items
  • Sets item count to the new size
  • Existing items are preserved if new size is larger
  • Items beyond new size are lost if new size is smaller

Example:

// Pre-allocate space for 100 items
GS_List<float> particles;
particles.SetNumItems(100);

// All 100 slots now exist (but may contain garbage)
for (int i = 0; i < 100; i++)
{
    particles.SetItem(i, 0.0f);
}

Status Methods

IsEmpty()

File: gs_list.h (inline)

Checks if the list contains no items.

Signature:

inline BOOL IsEmpty()

Parameters:

  • None

Returns:

  • TRUE - List has no items
  • FALSE - List contains one or more items

Example:

GS_List<int> numbers;

if (numbers.IsEmpty())
{
    printf("List is empty\n");
}

numbers.AddItem(10);

if (!numbers.IsEmpty())
{
    printf("List has items\n");
}

Best Practices & Examples

Managing Game Objects

// Store pointers to enemy objects
GS_List<Enemy*> enemies;

// Add enemies
for (int i = 0; i < 10; i++)
{
    Enemy* pEnemy = new Enemy();
    pEnemy->SetPosition(rand() % 800, rand() % 600);
    enemies.AddItem(pEnemy);
}

// Update all enemies
for (int i = 0; i < enemies.GetNumItems(); i++)
{
    enemies[i]->Update();
}

// Remove dead enemies (iterate backwards!)
for (int i = enemies.GetNumItems() - 1; i >= 0; i--)
{
    if (enemies[i]->IsDead())
    {
        delete enemies[i];
        enemies.RemoveIndex(i);
    }
}

// Clean up remaining enemies
for (int i = 0; i < enemies.GetNumItems(); i++)
{
    delete enemies[i];
}
enemies.RemoveAllItems();

Audio Playlist Management

// Using GS_List with FMOD music modules
GS_List<FMUSIC_MODULE*> m_gsMusicList;

// Add music tracks
FMUSIC_MODULE* track1 = FMUSIC_LoadSong("music/track1.mod");
m_gsMusicList.AddItem(track1);

FMUSIC_MODULE* track2 = FMUSIC_LoadSong("music/track2.xm");
m_gsMusicList.AddItem(track2);

// Play a specific track
int trackIndex = 0;
if (trackIndex < m_gsMusicList.GetNumItems())
{
    FMUSIC_PlaySong(m_gsMusicList[trackIndex]);
}

// Clean up music
for (int i = 0; i < m_gsMusicList.GetNumItems(); i++)
{
    FMUSIC_FreeSong(m_gsMusicList[i]);
}
m_gsMusicList.RemoveAllItems();

Texture Management

// Manage texture resources
GS_List<GS_OGLTexture*> textures;

// Load textures
const char* files[] = { "sprite1.tga", "sprite2.tga", "sprite3.tga" };
for (int i = 0; i < 3; i++)
{
    GS_OGLTexture* pTex = new GS_OGLTexture();
    if (pTex->Create(files[i]))
    {
        textures.AddItem(pTex);
    }
    else
    {
        delete pTex;
    }
}

// Find a specific texture
GS_OGLTexture* pTarget = /* some texture */;
int index = textures.FindItem(pTarget);
if (index != -1)
{
    printf("Texture found at index %d\n", index);
}

// Clean up
for (int i = 0; i < textures.GetNumItems(); i++)
{
    delete textures[i];
}
textures.RemoveAllItems();

Storing Value Types

// Store player scores
GS_List<int> highScores;

// Add scores
highScores.AddItem(50000);
highScores.AddItem(75000);
highScores.AddItem(100000);

// Sort scores (simple bubble sort)
for (int i = 0; i < highScores.GetNumItems() - 1; i++)
{
    for (int j = 0; j < highScores.GetNumItems() - i - 1; j++)
    {
        if (highScores[j] < highScores[j + 1])
        {
            int temp = highScores[j];
            highScores.SetItem(j, highScores[j + 1]);
            highScores.SetItem(j + 1, temp);
        }
    }
}

// Display top 10 scores
for (int i = 0; i < min(10, highScores.GetNumItems()); i++)
{
    printf("%d. %d\n", i + 1, highScores[i]);
}

Important Tips

  • Iterate Backwards When Removing: When removing items during iteration, go backwards to avoid index shifting issues
  • Pointer Cleanup: Remember to delete objects before removing pointers from the list
  • Pre-allocation: Use constructor parameter or SetNumItems() to pre-allocate if you know the size
  • Bounds Checking: Always check indices before accessing items, especially with user input
  • Type Requirements: Stored type must support == operator for Find and Remove operations
  • Memory Efficiency: List grows by one item at a time; pre-allocate for better performance

Common Patterns

  • Resource Lists: Use for textures, sounds, sprites, fonts
  • Object Pools: Pre-allocate and reuse objects like bullets or particles
  • Dynamic Collections: Enemies, pickups, UI elements
  • Configuration Data: Level waypoints, spawn locations

⚠️ Memory Management Warning

GS_List only manages the memory for the array itself. If you store pointers to dynamically allocated objects, you are responsible for deleting those objects before removing them from the list or destroying the list. Failing to do so will cause memory leaks.
